## 3. Oversight Capabilities

### 3.1 Understanding the system
How can the overseer understand the system's capabilities and limitations?
- [ ] Training materials provided
- [ ] System documentation accessible
- [ ] Performance metrics dashboard available

### 3.2 Monitoring
How can the overseer detect anomalies, dysfunctions, or unexpected performance?
- [ ] Real-time monitoring interface
- [ ] Alerting system for anomalous outputs
- [ ] Audit log accessible to overseer

### 3.3 Interpretation
How can the overseer correctly interpret the system's output?
- [ ] Confidence scores displayed
- [ ] Explanation/reasoning provided
- [ ] Context and limitations shown alongside output

### 3.4 Override and intervention
- [ ] Overseer can decide not to use the system's output
- [ ] Overseer can override or reverse the system's output
- [ ] "Stop" button or equivalent emergency interruption mechanism exists
- [ ] Fallback procedure documented for system failure

---

## 4. Automation Bias Mitigation

Measures to prevent the overseer from over-relying on the AI system's output:

- [ ] Training on automation bias risks
- [ ] System does not present outputs as definitive/authoritative
- [ ] Regular calibration exercises
- [ ] Dual-review requirements for high-stakes decisions
